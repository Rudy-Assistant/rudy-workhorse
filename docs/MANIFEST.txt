FEATURES & CAPABILITIES
================================================================================

INTELLIGENCE GATHERING
  • Monitors 20+ RSS feeds covering:
    - AI/ML tools (Anthropic Claude, OpenAI GPT, Meta AI, DeepMind)
    - Image generation (DALL-E, Stable Diffusion, FLUX)
    - Video generation (Sora, Runway, generative video)
    - Music generation (Suno, Udio)
    - Privacy & security (VPN, encryption, anonymity tools)
    - Legal technology (contract automation, legal AI, due diligence)
    - Automation (RPA, Zapier, n8n, smart home)

RELEVANCE SCORING
  • Keyword-based matching system
  • 7 content categories with multipliers:
    - AI/ML: 2.5x (highest priority)
    - Image Generation: 2.0x
    - Video Generation: 2.0x
    - Music Generation: 1.8x
    - Legal Tech: 1.8x
    - Privacy: 1.5x
    - Automation: 1.5x
  • Recency boost (newer items score higher)
  • Combined score = base × multiplier × recency

REPORT GENERATION
  • JSON format: Full structured data with scores and metadata
  • Markdown format: Human-readable digest with categories
  • Categorized grouping and sorting
  • Timestamp and total item count metadata

DEDUPLICATION
  • URL-based caching (research-feed-cache.json)
  • Normalized URL comparison (lowercase, no trailing slash)
  • Prevents duplicate reports across runs
  • Auto-maintained (grows ~100-200 URLs/day)

OPERATIONAL MODES
  • Quick mode: 30-60 seconds (5 feeds, 5 items each)
  • Full mode: 5-15 minutes (20+ feeds, comprehensive)
  • Debug mode: Verbose logging for troubleshooting

FEED MANAGEMENT
  • Add/remove feeds: workhorse-subscribe.py
  • Validate connectivity: Automatic health checks
  • Export/import: Backup and restore configurations
  • Configuration file: research-feeds.json

SYSTEM REQUIREMENTS
================================================================================

SOFTWARE
  • Python 3.10 or higher
  • Windows 11 (tested on The Workhorse)
  • Task Scheduler (for automation)

PYTHON PACKAGES (auto-installed by setup)
  • requests (HTTP fetching)
  • beautifulsoup4 (HTML parsing)
  • feedparser (RSS/Atom parsing)

HARDWARE
  • Network: Outbound HTTPS (port 443) access
  • Bandwidth: ~10 MB per full run, ~1 MB per quick run
  • Disk: ~50 MB for all logs, cache, and reports

NETWORK
  • Outbound HTTPS to RSS feed hosts (no proxy required)
  • ~10 MB bandwidth per full run
  • Sequential feed fetching (not parallel)

TROUBLESHOOTING REFERENCE
================================================================================

PROBLEM: ModuleNotFoundError for feedparser/requests/beautifulsoup4
  SOLUTION: pip install feedparser requests beautifulsoup4

PROBLEM: No items in reports
  ACTION 1: python workhorse-subscribe.py validate
  ACTION 2: python workhorse-research-feed.py --debug
  ACTION 3: rm rudy-logs\research-feed-cache.json (clear cache)

PROBLEM: Script execution fails
  ACTION: python validate-research-feed.py (comprehensive check)

PROBLEM: Scheduled task not running
  ACTION 1: Verify in Task Scheduler
  ACTION 2: Check task history for errors
  ACTION 3: Verify python.exe path: where python

PROBLEM: Feed timeouts
  SOLUTION: Use --quick mode or increase REQUEST_TIMEOUT

See RESEARCH-FEED-README.md for comprehensive troubleshooting.
