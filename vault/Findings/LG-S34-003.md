# LG-S34-003: Desktop Commander read_file metadata-only bug

> [!bug] LOW Finding
> DC `read_file` returns metadata-only JSON instead of file contents. Recurring across all sessions.

## Context

Desktop Commander's `read_file` tool intermittently returns only file metadata (filename, path, type) without actual content. This affects all file reads in Cowork/CLI sessions.

## Workaround

Use `robin_read_file.py`, Windows MCP Shell (`Get-Content`), or write a Python helper to `rudy-data/` and execute via `start_process`. **NEVER call `read_file` repeatedly hoping it works.**

## Root Cause

Unknown — likely a Desktop Commander MCP bug. Filed as recurring.
