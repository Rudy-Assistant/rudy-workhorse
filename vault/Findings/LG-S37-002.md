# LG-S37-002: bridge-heartbeat.json not created on first boot

> [!warning] MEDIUM Finding
> `bridge-heartbeat.json` is not created by `bridge_runner.py` on first boot, causing FoxGate liveness check to fail until first heartbeat cycle completes.

## Context

When bridge_runner.py starts fresh (no existing heartbeat file), the FoxGate protocol's Step 1 Item 1 (Robin liveness check) will fail because the file doesn't exist yet. The heartbeat is only written after the first iteration completes.

## Proposed Fix

Write an initial heartbeat immediately on startup before entering the main loop.
