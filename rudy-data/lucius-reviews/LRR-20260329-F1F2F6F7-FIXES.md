## Critical Issues

| # | Line | Issue | Severity |
|---|------|-------|----------|
| 1 | ~220 | **Lock race condition:** Between checking `LOCK_FILE.exists()` and writing it, another process could also pass the check and write. This is a TOCTOU (time-of-check/time-of-use) race. | ЁЯЯб Medium |
| 2 | ~306 | **URL sanitization strips query params:** The allowlist `_SAFE_CHARS` doesn't include `?`, `=`, `&`, `#`, `%`, `+` тАФ all needed for valid URLs. A URL like `https://github.com/search?q=test` would become `https://github.com/searchqtest`. | ЁЯФ┤ High |
| 3 | ~566 | **seed-deep doesn't pass force:** CLI `seed-deep` calls `seed_standard_nightwatch()` without `force=True`, so manual `seed-deep` could be blocked by cooldown even though manual `seed` isn't. | ЁЯЯб Medium |

## Suggestions

| # | Line | Suggestion | Category |
|---|------|------------|----------|
| 1 | ~220 | Use `os.open()` with `O_CREAT | O_EXCL` for atomic lock creation instead of exists-then-write. This eliminates the TOCTOU race. | Correctness |
| 2 | ~57 | Add URL-safe chars (`?=&#%+@~`) to a separate URL sanitizer or add a `url_mode` parameter to `_sanitize_metadata_string`. | Correctness |
| 3 | ~566 | Pass `force=True` in CLI `seed-deep` command too. | Correctness |
| 4 | ~330 | The `safe_paths` list is hardcoded. Consider making it a module-level constant or loading from config so it's easier to extend. | Maintainability |
| 5 | ~217 | `import os as _os` inside `_acquire_lock` is odd тАФ `os` is already available at module level (used elsewhere). Move to top-level import. | Style |
